Apache NiFi Cluster Automation

Deploying Scalable Data Flow with Ansible

What is Apache NiFi?

  • Visual data orchestration tool
  • Process and distribute data between systems
  • High-level control, real-time data flow
  • Built-in lineage and tracking (Data Provenance)

The Challenge: Clustering

  • Multi-node coordination
  • External or Embedded ZooKeeper management
  • Complex `nifi.properties` sync
  • Security (HTTPS/Users/Authorizers) setup

The Solution: This Playbook

A declarative approach to NiFi infrastructure.

  • Automated Java and NiFi installation
  • Embedded ZooKeeper auto-configuration
  • Dynamic State Management setup
  • Ansible-managed User & Policy XMLs

Playbook Overview


- name: Setup NiFi Cluster
  hosts: nifi
  become: yes
  roles:
    - common          # Java, dependencies
    - nifi            # Core Install & ZK
    - nifi_security   # Users & Policies

Embedded ZooKeeper Automation

The playbook handles the complex task of creating `zookeeper.properties` and the `myid` files for all cluster nodes automatically based on the Ansible inventory.

Declarative User Management

Users and initial admins are defined as variables, and the playbook generates the required `users.xml` and `authorizers.xml` for you.
